# Truther Contracts SDK — Go

Generated Go SDK for Truther protobuf contracts. **Do not edit files in this directory manually** — they are auto-generated by `buf generate`.

## Installation

```bash
go get github.com/vinirmbrozz/truther-contracts/gen/go
```

## How to add a new contract

1. Add or edit a `.proto` file under `proto/` at the repo root.
2. Run `buf generate` from the repo root — this regenerates `gen/go/`.
3. Commit both `proto/` and `gen/go/` changes in the same PR.
4. Update `gen/go/go.mod` if new proto imports require new Go dependencies.

The CI workflow (`.github/workflows/generate.yml`) re-runs `buf generate` and fails if the committed output is stale.

## How to use produce/consume in Go

The `serde` package (at `serde/` in this repo) provides Confluent Schema Registry-aware Kafka serialization using the wire format defined in the [Confluent SR SPEC](../../docs/confluent-sr-serde-spec.md).

### Setup

```go
import (
    "github.com/vinirmbrozz/truther-contracts/serde"
    pb "github.com/vinirmbrozz/truther-contracts/gen/go"
)

// Create serde from environment variables (SCHEMA_REGISTRY_URL must be set).
// Alternatively: serde.NewWithConfig(serde.Config{SRURL: "http://localhost:8081"})
s, err := serde.New()
if err != nil {
    log.Fatal(err)
}

// Register the Go type for each topic you will consume.
// Pass the actual .proto IDL text as the third arg for production SR;
// pass "" to derive a minimal identifier (fine for tests with a mock SR).
s.RegisterType("transactions", &pb.Transaction{}, protoIDL)
```

### Producing a Kafka message

```go
tx := &pb.Transaction{
    TransactionAmount: "499.99",
    FinalDecision:     "APPROVED",
    PredictiveAnalyzer: &pb.PredictiveAnalyzer{
        IsAllowed:     true,
        Reason:        "OK",
        CardId:        "card-123",
        UserId:        "user-456",
        WalletAddress: "0xABCD...",
        Allowance:     "5000.00",
        TransactionId: "txn-789",
    },
}

// framedBytes is the Confluent-framed value ready for your Kafka producer.
framedBytes, err := s.Produce("transactions", tx)
if err != nil {
    log.Fatal(err)
}
```

### Consuming a Kafka message

```go
// data is the raw Kafka message value bytes from the consumer.
msg, err := s.Consume("transactions", data)
if err != nil {
    // Invalid magic byte, unknown schema ID, or corrupt payload — route to DLQ.
    log.Printf("DLQ transactions-dead-letter: %v", err)
    return
}

tx := msg.(*pb.Transaction)
```

## Wire format

Every Kafka message value uses the **Confluent envelope**:

```
[0x00] [schema_id_be4] [0x00] [proto3_binary_payload]
```

See [docs/confluent-sr-serde-spec.md](../../docs/confluent-sr-serde-spec.md) for the full specification.

## Environment variables

| Variable | Required | Description |
|---|---|---|
| `SCHEMA_REGISTRY_URL` | Yes | SR base URL — e.g. `http://localhost:8081` |
| `SCHEMA_REGISTRY_API_KEY` | No | SR API key (authenticated deployments) |
| `SCHEMA_REGISTRY_API_SECRET` | No | SR API secret |

## Local development

Start the local Schema Registry via Docker Compose:

```bash
docker-compose up -d schema-registry
export SCHEMA_REGISTRY_URL=http://localhost:8081
```

Run the serde tests:

```bash
cd serde
make test   # runs go mod tidy + go test -v -race ./...
```
